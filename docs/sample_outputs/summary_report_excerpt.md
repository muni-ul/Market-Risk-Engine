# Summary Report Excerpt

This is a representative excerpt from `results/demo_run/summary_report.md`. The real values are generated by the CLI run.

```markdown
# PyRiskLab Run Summary

Run name: `demo_run`
Seed: `42`
Output directory: `results/demo_run`

## Simulation Only

This is a local simulation only. It does not use live market data, place real trades, connect to a brokerage, provide investment advice, or make profitability claims.

## Market Simulation

- Initial underlying price: $100.00
- Final underlying price: generated from the seeded synthetic path
- Generated steps: controlled by `configs/demo.yaml`
- Model: geometric Brownian motion with synthetic data
- Drift assumption: from config
- Volatility assumption: from config

## Option Contract

- Symbol: configured option symbol
- Type: call or put
- Strike: configured strike
- Initial days to expiry: configured expiry horizon
- Initial model price: generated by Black-Scholes
- Final model price: generated by Black-Scholes

## Greeks

- Final delta: generated from the last row of `greeks_history.csv`
- Final gamma: generated from the last row of `greeks_history.csv`
- Final vega: generated from the last row of `greeks_history.csv`
- Final theta: generated from the last row of `greeks_history.csv`
- Final rho: generated from the last row of `greeks_history.csv`

## Strategy Signals

- Strategy: generated from `strategy.name`
- Buy when delta is below: generated from `strategy.buy_delta_below`
- Sell when delta is above: generated from `strategy.sell_delta_above`
- Trade quantity: generated from `strategy.trade_quantity`
- Minimum steps between trades: generated from `strategy.min_steps_between_trades`
- Total signal rows: generated by the strategy module
- Proposed simulated orders: generated by the order builder
- Approved simulated orders: generated by risk validation
- Blocked simulated orders: generated by risk validation
- Skipped simulated orders: generated when fake execution is disabled

## Portfolio Results

- Starting cash: configured risk starting cash
- Number of simulated trades: generated by fake execution
- Final cash: generated from portfolio accounting
- Final position quantity: generated from portfolio accounting
- Final realized P&L: generated from portfolio accounting
- Final unrealized P&L: generated from portfolio accounting
- Peak portfolio value: generated from portfolio accounting
- Final portfolio value: generated from portfolio accounting
- Max drawdown: generated from portfolio history
- Risk events: generated by the risk manager

## Fake Execution

- Enabled: generated from `execution.enabled`
- Fill model: generated from `execution.fill_model`
- Commission per contract: generated from `execution.commission_per_contract`
- Contract multiplier: generated from `execution.contract_multiplier`
- Result: generated from fake execution state, or reports that fake execution was disabled

## Risk Events

- Max position quantity: generated from `risk.max_position_quantity`
- Max trade notional: generated from `risk.max_trade_notional`
- Max drawdown: generated from `risk.max_drawdown_pct`
- Max loss: generated from `risk.max_loss_pct`
- Stop trading on breach: generated from `risk.stop_trading_on_breach`
- Result: generated from risk manager state; empty state says no risk events were triggered

## Benchmark

Vectorized NumPy pricing ran `<generated speedup>`x faster than the Python loop on this machine.

- Prices compared: generated from `benchmark.num_prices`
- Python loop runtime: generated with `time.perf_counter()`
- NumPy vectorized runtime: generated with `time.perf_counter()`
- Max absolute error vs loop: generated from the equivalence check
- Numerical equivalence check: passed

Benchmark results vary by hardware, Python version, and input size.

Benchmark skipped because benchmark.enabled is false.

## Limitations

PyRiskLab uses synthetic paths and a simplified fake execution model. Results are for software demonstration only.
```
